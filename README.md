# stash-server

Самостоятельно размещаемый менеджер секретов с RESTful HTTP API и хранилищем на базе PostgreSQL.

## Обзор

`stash-server` — основной серверный компонент экосистемы Stash. Он предоставляет структурированный API для управления **хранилищами (stash)** и **секретами**, организованными по пользователям. Сервер предназначен для локального развёртывания и интеграции с инфраструктурными инструментами: Terraform, CLI-клиентами и Go SDK.

## Возможности

- REST API для создания пользователей, хранилищ и секретов
- Аутентификация на основе JWT
- Интеграция с `stash-sdk`, `terraform-provider-stash` и `stashctl`

## Архитектура

Проект имеет [гексагональную архитектуру](https://ru.wikipedia.org/wiki/%D0%93%D0%B5%D0%BA%D1%81%D0%B0%D0%B3%D0%BE%D0%BD%D0%B0%D0%BB%D1%8C%D0%BD%D0%B0%D1%8F_%D0%B0%D1%80%D1%85%D0%B8%D1%82%D0%B5%D0%BA%D1%82%D1%83%D1%80%D0%B0)

Ресурсы организованы иерархически: каждый пользователь владеет одним или несколькими хранилищами, каждое хранилище содержит секреты (пары ключ-значение).

Владелец хранилища может открыть хранилище предоcтавив пароль от него и закрыть хранилище.

Владелец хранилища может добавлять других пользователей, которые будут иметь возможность считывать хначения секретов в случае если хранилище находиться в открытом состоянии.

## Быстрый старт

### Сборка с помощью just

```bash
git clone --depth 1 https://github.com/okunix/stash-server.git
just build
```

### Конфигурация

Скопируйте пример конфигурации и настройте под своё окружение:

```bash
cp config.example.yaml config.yaml
```

### Запуск

```bash
./bin/stash-server --config config.example.yml run
```

### Запуск с помощью Docker Compose

```bash
docker compose up
```

## Обзор API

Полная документация API доступна через Swagger UI по адресу `/swagger/index.html` при запущенном сервере.
